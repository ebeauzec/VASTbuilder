Solution Overview

Technical Solution Overview

Architecture — DASE Model

VAST Data's Disaggregated Shared-Everything (DASE) architecture separates stateless compute (C-nodes) from high-density NVMe storage enclosures (D-nodes) over a non-blocking NVMe-oF fabric.

Data Services

  • Protocol Support: NFSv3, NFSv4.1, SMB 3.x, S3, NVMe-oF/TCP
  • Data Reduction: Similarity Reduction + Global Deduplication + Compression
  • Erasure Coding: VAST 150+4 (2.6% overhead, 4-drive fault tolerance)
  • Snapshots: Instant, space-efficient, unlimited retention
  • Replication: Async/Sync DataSpace mirroring to on-prem or cloud

Client Mount Guide

Optimised mount commands for all configured protocols. Replace 10.100.20.100 with your assigned VIP address.

NFSv3 — Linux (High-Throughput)
Recommended for AI/ML training, HPC, and high-bandwidth workloads. Uses 8 TCP connections (nconnect) to saturate 100GbE client links.
mount -t nfs -o vers=3,proto=tcp,nconnect=8,rsize=1048576,wsize=1048576,hard,timeo=600,retrans=2 10.100.20.100:/data/ai_models /mnt/vast
NFSv4.1 — Linux (Secure / ACLs)
Use for workloads requiring fine-grained POSIX ACLs, file locking, or Kerberos authentication.
mount -t nfs -o vers=4.1,proto=tcp,nconnect=8,rsize=1048576,wsize=1048576,hard,timeo=600,retrans=2 10.100.20.100:/data/ai_models /mnt/vast-nfs4
SMB — Windows (PowerShell)
Maps a persistent network drive on Windows clients using SMB 3.x with NTLM or Kerberos authentication.
New-PSDrive -Name "V" -PSProvider FileSystem -Root "\\10.100.20.100\data_ai_models" -Persist -Credential (Get-Credential)
S3 — AWS CLI Configuration
Configure the AWS CLI to target the VAST S3-compatible endpoint. Tune concurrent requests for maximum throughput on large object workloads.
aws configure set default.s3.max_concurrent_requests 32 && aws configure set default.s3.multipart_threshold 64MB && aws s3 ls s3://vast-bucket/ --endpoint-url http://10.100.20.100:9000
Kubernetes — PVC YAML
Create a ReadWriteMany PersistentVolumeClaim backed by the VAST CSI driver.
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: vast-pvc
  namespace: default
spec:
  storageClassName: vast-storageclass
  accessModes:
    - ReadWriteMany
  resources:
    requests:
      storage: 10Ti

Persistent Mount (fstab)

10.100.20.100:/data/ai_models /mnt/vast nfs vers=3,proto=tcp,nconnect=8,rsize=1048576,wsize=1048576,hard,timeo=600,_netdev 0 0
